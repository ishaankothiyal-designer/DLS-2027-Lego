## Use this skill for

- visual refreshes
- Storybook UI improvement
- hierarchy and composition fixes
- premium-feeling documentation pages
- calming noisy interfaces
- making product UI feel more intentional without breaking the token contract

## Workflow

1. Diagnose the failure mode first: flat hierarchy, noisy styling, weak spacing, excessive density, or weak brand expression.
2. Choose the dominant visual quality needed for the page: precision, operational clarity, approachable hierarchy, or warm utility.
3. Fix composition before styling:
   - establish the content spine
   - strengthen grouping
   - create one dominant zone
   - reduce repeated container treatments
4. Run a legibility pass:
   - confirm readable line lengths
   - reduce thin or fragile type choices
   - verify contrast and state clarity
   - ensure dense regions have explicit density intent
5. Apply the system layers in order:
   - `Semantic` for meaning and contrast
   - `Theme` for brand tone
   - `Typography` for hierarchy and texture
   - `Misc` spacing and size foundations for rhythm
6. Limit flourish:
   - one main surface treatment
   - one secondary treatment
   - one accent strategy
7. Review the result with the checklist in [../../memory/visual-composition-principles.md](../../memory/visual-composition-principles.md).

## Storybook guidance

- Make docs feel curated, not merely generated.
- Prefer editorial structure over dashboard sameness.
- Reduce chip, badge, and accent overuse in headers.
- Use backgrounds and gradients once at the page level, not repeatedly at the card level.
- Optimize token views for comparison, scanability, and trust.
- Give reading-heavy pages calmer density than token-comparison pages.
- Use max-width and line-length restraint so docs feel authored, not stretched.

## Guardrails

- Do not fight the token system with local one-off colors or spacing values.
- Do not make every section look important.
- Do not confuse minimalism with emptiness.
- Do not rely on blur, glass, or large radii to create "premium."
- Do not add expression before hierarchy is working.
- Do not allow fragile contrast, tiny captions, or thin text to masquerade as sophistication.
- Do not rely on color alone for state meaning.

## Success criteria

- The page reads clearly in five seconds.
- There is one obvious focal region.
- The interface feels calmer without losing information.
- Brand expression is present but controlled.
- Text remains comfortable and legible across realistic viewing conditions.
- Layout rhythm holds up without decorative treatments.
- A designer would recognize intention, not just cleanliness.
