# Component Contracts

This directory contains the first active component contracts built on top of the current token foundations.

## Purpose

These files do not yet represent a serialized token export.

Instead, they define:
- the slots a component needs
- the states and variants it supports
- which existing token layers it should consume
- how future component tokens should be shaped

## Current Contracts

- [button.md](./button.md)
- [text-field.md](./text-field.md)
- [badge.md](./badge.md)

## Working Rule

If implementation begins before a formal component-token collection exists, these contracts are the source of truth for which Semantic, Theme, Typography, Utility, and Misc foundations should be used.
